Отчёт по воспроизведению результатов статьи о PINNs

Цель работы состояла в том, чтобы воспроизвести основные нейросетевые эксперименты из статьи о Physics-Informed Neural Networks, проверить их метрики, прогнать таблицы чувствительности и понять, какие результаты действительно подтверждаются, а какие нет.

Работа была выполнена в несколько этапов: сначала через общую репликацию на PyTorch, затем через контрольные прогоны на TensorFlow 2 в compat-режиме, а после этого через отдельный legacy-стек WSL + Python 3.7 + TensorFlow 1.15 + tf.contrib, максимально близкий к эпохе статьи.

1. Что воспроизводилось

  • Continuous Burgers, paper error 6.7e-4.
  • Continuous Schrodinger, paper error 1.97e-3.
  • Discrete Burgers, paper error 8.2e-4.
  • Allen-Cahn, paper error 6.99e-3.
  • Таблицы чувствительности Continuous Burgers: Table 1 и Table 2.
  • Таблицы чувствительности Discrete Burgers: Table 3 и Table 4.

2. История работы

2.1. Репликация на PyTorch

Сначала был собран единый пайплайн на PyTorch + SciPy для всех 4 экспериментов, multi-seed прогонов и таблиц чувствительности. Это дало базовую точку сравнения и позволило быстро увидеть, какие задачи воспроизводятся легко, а какие нет.

2.2. Проверка влияния GPU

После первых результатов стало видно, что дискретные эксперименты сходятся хорошо, а continuous-часть, особенно continuous Burgers, заметно расходится с paper. Была проверена гипотеза, что проблема в CPU. Для этого была восстановлена CUDA-среда и выполнен отдельный GPU-прогон. Он дал результат хуже CPU, так что простое включение GPU проблему не решило.

2.3. Проверка TensorFlow 2 / compat.v1

Следующим шагом была проверка гипотезы, что критично не только устройство, но и сам optimizer path TensorFlow. Для этого был собран контрольный runner для continuous Burgers на TF2 + compat.v1. Были проверены native Windows CPU, WSL CPU и WSL GPU.

Именно на этом этапе стало видно, что стек действительно влияет, но различия между фреймворками не объясняют картину полностью: TensorFlow улучшал некоторые результаты, но не гарантировал совпадение со статьёй.

2.4. Подъём legacy-стека под настоящий TensorFlow 1.15

После этого был развёрнут отдельный WSL-стек под Python 3.7.12 и TensorFlow 1.15.0 с tf.contrib.opt.ScipyOptimizerInterface. Для него были сделаны отдельные runner'ы для continuous Burgers, таблиц Table 1-2 и continuous Schrodinger. Для длинного sweep по таблицам был добавлен checkpoint/resume по ячейкам, чтобы можно было безопасно продолжать после прерываний.

4.3. Continuous Schrodinger

Стек / запуск Ошибка Отношение к paper Комментарий
PyTorch budgeted CPU mean 2.6973e-3 1.37x Близко, но запуск был урезан по бюджету.

Полный official TF1 baseline занял около 6185 s на Adam и 787 s на L-BFGS, то есть примерно 116 минут суммарно. Именно здесь стек сыграл ключевую роль и дал paper-level результат.

4.4. Discrete Burgers

По 3 seed получено mean 5.8593e-4 против paper 8.2e-4. Таблицы тоже близки: среднее абсолютное расхождение Table 3 = 0.02166, Table 4 = 0.01036. Это хороший уровень совпадения.

4.5. Allen-Cahn

По 2 seed получено mean 6.7439e-3 против paper 6.99e-3. То есть эксперимент воспроизводится практически точно по среднему уровню ошибки.

6. Мои рассуждения и интерпретация

6.1. Что удалось установить надёжно

  1. Проблема не в том, что PyTorch "в принципе не умеет" PINNs. Если бы это было так, дискретные задачи тоже ломались бы, но этого не произошло.
  2. Проблема не сводится к простому включению GPU. Для Continuous Burgers GPU-пути оказались либо хуже, либо явно нестабильны.
  3. Для Continuous Schrodinger стек действительно критичен. Именно legacy TF1 дал paper-level результат.
  4. Для Continuous Burgers даже правильный стек проблему не решил. Значит, корень расхождения глубже, чем просто замена фреймворка.

6.2. Наиболее вероятные причины расхождения по Continuous Burgers

  • Очень высокая чувствительность к seed, LHS-sampling и траектории L-BFGS.
  • Возможное отличие GitHub-репозитория от точного внутреннего code path, который дал числа статьи.
  • Существенная разница между современной вычислительной средой и средой эпохи статьи.
  • Возможность того, что значение статьи соответствует особенно удачному запуску.

6.3. Почему Schrodinger сошёлся, а Burgers нет

Это главный научно-интересный результат всей работы. Если бы все continuous-модели ломались одинаково, можно было бы винить стек целиком. Но вышло иначе: Schrodinger в official TF1 сошёлся, а Burgers не сошёлся даже там. Значит, воспроизводимость статьи неоднородна и зависит от конкретной постановки задачи.

Гипотеза "всё не сходится из-за неправильного стека" оказалась слишком простой. Для Schrodinger стек действительно оказался ключевым, а для Burgers даже правильный стек проблему не решил.

7. Итог

На текущем этапе наиболее честный вывод такой: статья воспроизводится частично. Все дискретные эксперименты подтверждаются, continuous Schrodinger подтверждается в правильном legacy TF1-стеке, а continuous Burgers и его sensitivity-таблицы не подтверждаются до paper-level значений.

Основные артефакты отчёта: результаты в каталоге results/, а runner'ы и вспомогательные скрипты лежат в корне проекта.
